### Key Workflow Behavior Patterns

**Sequential Execution:**
- Steps execute one after another
- Output of one step becomes input of next
- Example: Basic Summarization ‚Üí Multi-Level Citation Summary

**Parallel Execution:**
- Multiple steps execute simultaneously
- Example: Context gathering from files, MCP, RAG in parallel

**Conditional Execution:**
- Steps execute based on conditions
- Example: Web search only if RAG results insufficient

**Loop Execution:**
- Steps repeat based on conditions
- Example: Iterative detail reduction until line count target met

**Nested Workflow Invocation:**
- Workflows call other workflows
- Example: Documentation Generation calls Basic Summarization

### File Creation Patterns

Each workflow documents file creation patterns:
- **Output paths** - Where files are created
- **Content structure** - What goes in each file
- **Format** - Markdown, JSON, code, etc.
- **Backup** - Whether backups are created

### Error Handling Patterns

Each workflow includes:
- **onFailure** action
- **maxRetries** limit
- **fallback** strategy
